1. Introduction

In this variant the basic evaluator is re-written in continuation passing style. Note that this variant isn't given in (Wadler, 1995) but is discussed in (Walder, 1992).

Again, the term structure of the language is the same: 

#lang racket
(provide (all-defined-out))

(require eopl/eopl)
(define-datatype term term?
  [con (int integer?)]
  [div (num term?) (den term?)])

(define :
  (lambda (f v1 v2)
    ((f v1) v2)))

2. Computation with continuations

The value domain remains as integers. A continuation maps values to answers. We choose the domain of answers to be a disjoint union of values (integers) and messages (for exceptions). (We could have used define-datatype instead.) A computation maps continuations to answers. 

;;; Semantic Domains
(define msg? string?)
(define value? integer?)

;;; type answer? = msg? + value?
(define answer? (or/c msg? value?))

;;; type continuation? = value? -> answer?
(define continuation? (value? . -> . answer?))

;;; type computation? = continuation? -> answer?
(define computation? (continuation? . -> . answer?))

3. Evaluator

Here is the evaluator. Notice that the eval-term is in CPS but its arguments are curried: the first (curried) argument is the term t and the second curried argument c is the continuation. 

;;; eval-term :  term? -> computation?
(define eval-term
  (lambda (t)
    (lambda (c)
      (cases term t
        [con (int) (c int)]
        [div (num den)
          (: eval-term num
            (lambda (n)
              (: eval-term den
                (lambda (d)
                  (if (= d 0)
                    (raise "divide by zero")
                    (c (quotient n d)))))))]))))

4. Error and top-level continuations

The continuation raise takes a message and returns it. The continuation tc takes a value and returns it. The invocation of these two functions will be so arranged such that their answers are always returned at top level. The run function takes a term and evaluates it against the top level continuation. 

;;; raise : msg? -> answer?
(define raise
  (lambda (msg)
    msg))

;;; tc : value? -> answer?
(define tc
  (lambda (v)
    v))

;;; run : term? -> answer?
(define run
  (lambda (t)
    (: eval-term t tc)))

5. Unit Tests

;;; Testing

;;; Racket's unit testing framework
(require rackunit)

(check-equal? (run (con 5)) 5 "run: 5 test")
(check-equal? (run (div (con 5) (con 3)))  1 "run: 5 test")
(check-equal? (run (div (con 5) (con 0))) "divide by zero" "run: divide by zero test1")
(check-equal? (run (div (con 7) (div (con 5) (con 0)))) "divide by zero" "run: divide by zero test2")

6. Summary

Answers are either (error) messages or (integer) values. Continuations map values to answers. Computations are functions that map continuations to answers. The evaluator maps terms to computations.
