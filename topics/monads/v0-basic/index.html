2. Term data type

The examples will all be based on a simple term language, which consist of integer constants and division operations. The following BNF captures the abstract syntax of terms. 

term? ::= (con integer?) |  (div (num term?) (den term?))

This can be turned into a scheme definition using define-datatype. 

(define-datatype term term?
  [con (int integer?)]
  [div (num term?) (den term?)])

3. Evaluator

The basic evaluator is simple. It takes a term and returns an integer. The key step towards understanding monads is to make the leap from integers to values and from values to computations. In the simple setting of the basic interpreter, the computations are just values, which are integers. 

(define value? integer?)
(define computation? value?)

;;; eval-term : term? -> computation?
(define eval-term
  (lambda (t)
    (cases term t
      [con (int) int]
      [div (num den)
        (let ([n (eval-term num)]
              [d (eval-term den)])
          (quotient n d))])))

4. Unit Tests

;;; Testing

;;; Racket's unit testing framework
(require rackunit)

(check-equal? (eval-term (con 5)) 5 "eval-term: 5 test")
(check-equal? (eval-term (div (con 5) (con 3))) 1 "eval-term: 5 test")
(check-exn exn? (lambda () (eval-term (div (con 5) (con 0)))) "divide by zero exception")

Notice that the last test case results in an exception that is not caught by eval-term.

5. Summary

Terms are either constant integers or division operations. The interpreter maps terms to integers.
