1. Introduction

The evaluator with state, recall that computations were state transformers, and that evaluators mapped terms to computations. In the monadic version, we define the unit and composition operators to support this view. Again, this is unchanged from before. 

#lang racket
(provide (all-defined-out))

(require eopl/eopl)

(define-datatype term term?
  [con (int integer?)]
  [div (num term?) (den term?)])

2. Stateful computations

The semantic domains remain the same: 

;;; Semantic Domains
(define nat?
  (and/c integer? (>=/c 0)))
(define state? nat?)
(define value? integer?)
(define  computation? (state? . -> . (list/c value? state?)))

3. Defining the unit operator

The unit operation takes a value and returns a computation. The computation takes a state and simply pairs the value with that state. 

;;; unit :  value? -> computation?
(define unit
  (lambda (a)
    (lambda (s)
      (list a s))))

4. Defining the composition operator o.

The composition operator o takes a computation m and a receiver r and returns a computation. This computation takes a state s, runs the computation m with s, giving a value v1 and s1. These are then passed to the receiver k to return the final result consisting of tuple v2 and s2. 

;;; o : computation? -> [(value? -> computation?) -> computation?]
(define o
  (lambda (m)
    (lambda (k)
      (lambda (s)
        (match-let ([(list v1 s1) (m s)])
          (match-let ([(list v2 s2) (: k v1 s1)])
            (list v2 s2)))))))

5. Counting the number of divisions as computation

We construct a function tick that represents the computation of incrementing the number of divisions. Notice that the value part of the result of the computation is arbitrarily defined to be zero; it could be anything because, as we shall see, this value is ignored. 

;;; tick : computation?
(define tick
  (lambda (s)
    (list 0  (+ s 1))))

6. Defining the evaluator

The evaluator is given below. Notice the three compositions, with the tick computation 'happening' before the quotient operation. 

;;; eval-term : term? -> monad?
(define eval-term
  (lambda (t)
    (cases term t
      [con (int) (unit int)]
      [div (num den)
        (: o (eval-term num)
           (lambda (n)
             (: o (eval-term den)
               (lambda (d)
                 (: o tick
                   (lambda (ignore)
                     (unit (quotient n d))))))))])))

That's it, we're done!

7. Unit tests

;;; Testing

;;; Racket's unit testing framework
(require rackunit)

(check-equal? (: eval-term (con 5) 0) (list 5 0) ": eval-term 5 0 test")
(check-equal? (: eval-term (div (con 5) (con 3)) 0) (list 1 1) ": eval-term  (div 5 3) 0 test")
(check-equal? (: eval-term (div (con 7) (div (con 5) (con 3))) 0) (list 7 2) ": eval-term  (div 7 (div 5 3)) 0 test")

8. Summary

A computation is a function that maps a state to a tuple consisting of a value and a state. The interpreter maps terms to computations. The monadic operators unit and o (composition) are defined in such a manner so as to hide the book keeping with the state. Incrementing the state is captured explicitly as a computation.
