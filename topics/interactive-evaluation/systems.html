1. Introduction

We introduce the notion of a transition system and see how transition systems may be used to model various notions related to terms and subterms.

2. (Transition) Systems

Informally, a transition system is a labeled directed graph where each vertex is associated with an observation. The vertex is called a state. A view function maps the state to observables. The system's state changes, or`transits', due the influence of actions. The change in state due to the action is governed by a fixed set of rules, collectively called the dynamics of the system. The dynamics is specified by a transition relation. When a system is in a given state and is acted upon by an action, the transition relation specifies the set of possible states to which the system may transit.

An interactive transition system is a transition system in which the actions are provided by a human agent and the observations are also directed at a human agent.

2.1. Formal Definition of a transition system

A transition system (specification) \(S\) is a tuple \(\pair{X, X^0, U, \xto{}{S}, Y, h}\) where

  1. \(X\), the state space is a set of states.
  2. \(X^0\subseteq X\) is the set of initial states of the system.
  3. \(U\), the action space, is a set of actions.
  4. \(\xto{}{} \subseteq X\times U\times X\), is the transition relation, or dynamics of the system. The element \((x,u,x')\) of the transition relation is called a transition and is written \(x\xto{u}{}x'\).
  5. \(Y\), the observation space, is a set of observables and its elements are called observations.
  6. \(h\), the view, is a function mapping a state to an observation.

Sometimes, we add a subscript to the elements to emphasize that the components belong to a particular system, e.g., \(X_S\), \(Y_S\), etc.

Systems are useful for modelling a variety of phenomena and mechanisms. It is easy to see that finite automata, both deterministic and non-deterministic, are systems, as are pushdown automata and Turing machines. (Exercise: redefine each of these automata as systems.)

2.2. State properties

If \(x\xto{u}{}x'\), then \(x'\) is called a \(u\)-successor of \(x\). \(x'\) is a successor of \(x\), written \(x\xto{}{}x'\), if there exists a \(u\) such that \(x'\) is a \(u\)-successor of \(x\). We write \(x\xto{u}{}\) to mean that \(x\) has a \(u\)-successor and \(x\not\!\!\xto{u}{}\) otherwise. \(x\) is a blocking or terminal state if \(x\) has no successor, otherwise it is non-blocking. We write \(x\not\!\!\xto{}{}\) to indicate that \(x\) is blocking and \(x\xto{}{}\) to indicate that it is non-blocking.

2.3. System Properties

A system is transparent if \(Y=X\) and \(h\) is the identity function. When a system is transparent, we omit \(Y\) and \(h\) in its specification. A system is deterministic if, for each state \(x\) and action \(u\), \(x\) transits to at most one state \(x'\) on \(u\). It is non-blocking if each of its states are non-blocking. It is autonomous if \(U\) is a singleton. The actual element of \(U\) is irrelevant, we omit it in the specification. We also write \(x\xto{}{}x'\), omitting the action. We elide mentioning the set of initial states when it is equal to the entire state space. A system is a mapcode system if it is (a) deterministic, (b) non-blocking, and (c) autonomous. A mapcode system is simply denoted \(\pair{X,F}\), where \(F:X\rightarrow X\) is an endofunction on \(X\).

2.4. Runs

Let \(\N\union \set{\omega}\) denote \(\N\) extended with the limit ordinal \(\omega\) (i.e., \(i < \omega\) for all \(i\in \N\)). Let \(x\in X_S\) and let \(n\in \N\union \set{\omega}\). A run originating from \(x\) of length \(n\) is a sequence \(x_i\xto{u_i}{S}x_{i+1}\), \(0\leq i < n\). The run is empty if \(n=0\), finite if \(n\in \N\), and infinite if \(n=\omega\). The destination of a finite run is \(x_{n}\). We say \(x_n\) is reachable from \(x_0\) if there is a run with origin \(x_0\) and destination \(x_n\). We say the action sequence \([u_i \ |\ 0 \leq i < n]\) steers \(x\) to \(x_{n+1}\). The action sequence is the just the concatenation of the labels of the edges on the path from \(x\) to \(x'\). The run is complete if it is either infinite or its destination is terminal.

A state is normalizable if there is some complete run originatinng from the state is finite. A state is strongly normalizable if every complete run originating from the state is finite.

3. Terms

Recall the notion of finite term trees (or just finite terms) introduced earlier. We revisit terms in a more general setting here, one that includes both finite and infinite terms.

3.1. Term systems

Given a term \(t\), the set \(T\) of inductive terms induces a deterministic transition system (also called \(T\)). \(T\)'s state space \(X_T\) consists of all finite terms, its action space is \(\N_1\), and its observation space is \(\Sigma\). We use \(t\) to denote a state. The view function \(h_T(t)\) maps the state (term) \(t\) to its head. Let \(t\) be the term \(f(t_1, \ldots, t_n)\) where \(f\in \Sigma\) and \(\alpha(f)=n\). Then \(t\xto{i}{T}{t_i}\) for each \(i\), \(1 i \leq n\).

The set \(T_{\omega}\) of co-inductive terms also induces a deterministic transition system (also called \(T_{\omega}\)). The states of \(T_{\omega}\) are the set of co-inductive terms. \(U_{T_{\omega}}\) and \(Y_{T_{\omega}}\) are the same as \(U_T\) and \(Y_T\) respectively. \(h_{T_{\omega}}(t) = \id{head}(t)\). If \(t=f(t_1, \ldots, t_n)\), then, \(t\xto{i}{T_{\omega}}t_i\) for each \(i\), \(0 < i \leq n\).

3.1.1. Example

Let \(\Sigma = \set{{\sf a}, {\sf g}}\) with \(\alpha({\sf a})=0\) and \(\alpha({\sf g})=1\). The transition system \(T_{\omega}\) contains a state \(t\) such that \[t = {\sf g}(t)\], in other words, \(h_{T_{\omega}}(t)={\sf g}\) and \(t\xto{1}{T_\omega}t\).

You may think of \(t\) representing a circular list. We have already seen an example of such `infinite' lists: streams.

In Racket, you can create such a circular list by using mutable pairs: 

(let ([x (mcons 'g #f)])
    (set-mcdr! x x)
    x)

Note that such circular terms are forbidden in \(T\), the transition system capturing inductive terms.

3.2. Transitive Closure and subterm

Let \(T\) be a transition system. We define the transitive system \(T^*\) as follows that is identical to \(T\) except in its action alphabet and transition relation. \(U_{T^*}=U_T^*\), \(x\xto{\pair{u_0, u_1, \ldots u_{m-1}}}{T^*}x'\) iff \([u_0, u_1, \ldots u_{m-1}]\) steers \(x\) to \(x'\) in \(T\). Note that \(x\xto{[\ ]}{T^*}x\) for each state \(x\in X_{T^*}\).

From now on, term will be short for co-inductive terms, i.e., elements in \(T_{\omega}\).

Let \(t\) be a term. The address space of \(t\), written \(\id{addr}(t)\), is the set of all \(\overline{u}\) such that \(x\xto{\overline{u}}{T_{\omega}^*}\). Note that \(\id{addr}(t)\) is prefix closed.

Let \(\overline{u}\in \id{addr}(t)\) and let \(t\xto{\overline{u}}{T_{\omega}^*}t'\). We say that \(t'\) is is the subterm of \(t\) at address (or position) \({\overline{u}}\). Because \(T_{\omega}^*\) is deterministic, the subterm of a term \(t\) at a position \(p\) in the address space of \(t\) is unique. The function \(\id{subterm_t}:\id{addr}(t) \rightarrow T_{\omega}\) maps a position \(p\) to the unique term \(t'\) such that \(t\xto{p}{T^*_{\omega}}t'\).

3.2.1. Example

Consider the following transition relations in \(T^*\):

\begin{align*} t_1 &= {\sf f}(t_2, t_3)\\ t_2 &= {\sf g}(t_4)\\ t_3 &= {\sf a}()\\ t_4 &= {\sf a}()\\ \end{align*}

\(\id{addr}(t_1)= \set{[\ ], [1], [2], [1\ 2]}\). The subterms \(t_4\) and \(t_3\) are located as positions \([1,\ 1]\) and \([2]\) respectively. The function \(\id{subterm}_{t_1}\) is given below

\begin{align*} [\ ] &\mapsto t_1\\ [1] &\mapsto t_2\\ [2] &\mapsto t_3\\ [1\ 1] &\mapsto t_4 \end{align*}

3.3. Replacement

Let \(t\) be a term and let \(t_p\) be the subterm of \(t\) at position \(p\). Let \(s\) be another term. The replacement of \(t\) at position \(p\) by term \(s\), denoted \(\id{replace}(t,p,s)\) is the term \(r\) defined as follows.

\begin{align*} \id{addr}(r) &= (\id{addr}(t) \setminus \set{a\in\id{addr}(t) |\ \id{prefix}(a)=p}) \\ &\union \set{p}\id{addr}(t_p) \end{align*} \begin{align*} \id{addr}(r) &= (\id{addr}(t) \setminus \set{a\in\id{addr}(t) |\ \id{prefix}(a)=p}) \\ &\union \set{p}\id{addr}(t_p) \end{align*}
