1. Introduction

In this section, we look at how to transform a program in register form into one that allows us to single-step through its execution.

2. Boilerplate   boilerplate

#lang racket
(provide (all-defined-out))

3. Starting point: Register variables

The starting point of our transformation is a program is the declaration of the register variables as in register form 

(define uninitialized '_)
(define *n* uninitialized)  ; input 
(define *answer* uninitialized) ; output
(define *i* uninitialized)  ; loop variable
(define *a* uninitialized)  ; accumulator

4. Code blocks

Consider the code blocks from the register transformation: init, loop and done. 

(define (init)
  (begin
    (set! *i* *n*)
    (set! *a* 1)
    (set! *answer* uninitialized)
    (goto loop)))

(define (done)
  (void))

(define (loop)
   (if (= 0 *i*)
       (begin
         (set! *answer* *a*)  
         (goto done))
       (begin
         (set! *a* (* *a* *i*))
         (set! *i* (sub1 *i*))
         (goto loop))))

5. Introduction of the program counter

The first step in the single-stepping transformation is to introduce a new register variable called the program counter. 

;;; *pc* = init|  loop | done
(define *pc* uninitialized)

(define (done?)
  (eq? *pc* done))

The program counter, when initialised, points to one of the code blocks.

6. Redefining goto

The second step is to redefine the goto function so that it sets the program counter, but does not invoke the code block at that program counter. 

(define (goto label)
  (set! *pc* label))

Notice each code block is essentially a sequence of assignment statements, with branching if needed. There are really no more calls. Transfer out of a code block is via goto, which is merely an assignment statement.

A step involves executing a single code block. 

(define (step)
  (*pc*))

7. Single-stepping and inspecting the program state

Inspecting the program state involves seeing the value of all register variables. The function show prints the values of all the registers. 

(define (show)
    (printf "pc: ~a~n" *pc*)
    (printf "n:  ~a~n" *n*)
    (printf "i: ~a~n" *i*)
    (printf "a: ~a~n" *a*)
    (printf "answer: ~a~n~n" *answer*))

(define (ss) ; executes a single step and displays the state
  (step)
  (show))

The program is now in single step form. The top-level interface functional interface is all that is left to define: 

(define (!/ss n)  ; interface function for single step form
  (begin
    (set! *n* n)
    (goto init)))
;; to continue evaluation, 
;; invoke (ss) repeatedly to single-step

The program is now a state machine with three locations: init, loop and done. At each location, the state of the program is determined by the values of the state variables.

8. Running till the end

run executes the program to completion by repeatedly single stepping until the done state is reached: 

(define (run)
   (if (done?)
       (printf "Final answer is ~a~n" *answer*)
      (begin
         (ss)
         (run))))

(define (! n)
  (!/ss n)
  (run))
