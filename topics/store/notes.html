1. Stores

1.1. Stores map locations to values

  • A store is a map from locations, or addresses to storable values. In our model, we assume the set of locations to be addressable, and we use the natural numbers to refer to addresses.
  • Consider set of statements in C:
+ int x = 5; {x -> =5= } 
+ x = x+1;   {x -> =6= }
  • id s are mapped or can be mapped not only to values but also to boxes of values (locations/addresses).
  • what happens internally for the above example?
Initially, int x = 5;
  Env: {x -> 1004}
    (where x is <id> and 1004 is address/location)
  Store:{1004 -> 5}
    (where 5 is value that <id> x has to store)
After eval of x = x+1 statement,
  Env: {x -> 1004}
  Store: {1004 -> 6}
Logically it is a new store but physically it is the same store.
  • Now our task is to define "Store" as ADT

1.2. Specification of ADT definition for store

  • Predicates
store? :: check for valid store
ref? (or) addr? (or) loc? :: check for valid location or address or reference
storable-value? :: check for values to be stored

  • Constructors

    store-size :: takes store and returns its size. 

store-size :: store? -> nat?

deref :: takes store and address and returns the value at that given address 

deref :: store? ref? -> storable-value?

setref :: takes store, address and value and returns the same store with the new value at the given address. 

setref :: store? ref? storable-value? -> store?

newref :: takes store and value as input and returns the extended store with the new value and the address of the location *of the new value 

newref :: store? storable-value? -> [store? ref?]

1.3. Evaluator with store

  • Earlier eval-ast would take a AST and env and give a value as answer
  • Now along with AST and env it takes a store and gives an expressible-value and a new/modified/same store
  • Language needs to update store
  • There are many ways to implement store, we choose the list notation, since it is easy to handle and a natural choice with Scheme
  • Whatever the choice of internal representation of the store the above-mentioned functionality should be part of it

1.3.1. Store Implementation

  • so our store is a list of any value
  • each value can be referenced using the index
(define store (list-of storable-value?))

(define ref (nat?))
  • implementation of constructors
new-ref? : store? storable-value? -> new-store? ref?
(define new-ref
   (lambda (s v)
     (list (cons v s) (length s))))
 
- adding to the front of the list is easier to implement, with less
  overhead, than adding at the end which would have been the logical
  thing to do
- the idea is now to keep track of the length of the store and
  remember to count back for the values

deref? : store? ref? -> storable-value?
(define deref
  (lambda (s r)
   (let ([n (length s)])
    (cond 
          [(< r n) (list-ref s (- n r 1))]
          [else (error "out of range")]))))

set-ref? : store? ref? storable-value? -> new-store?
(define set-ref
  (lambda (s r v)
    (let ([n (length s)])
    (cond 
          [(< r n) 
            ;build a new list same as 's' with the 'r'th location
  having value 'v'
            (replace s (- n r 1) v)]
          [else (error "out of range")]))))
  • currently, the check for valid ref is at the point of adding to the store
  • this can be avoided or responsibility can be shifted one level up for the check
  • dependent types
    • second argument is a function in terms of first argument
    • can be implemented by using currying: a way of using higher-order functions to achieve a more efficient programming
(define ref?
  (lambda (s)
    (lambda (i)
      (< i (length s)))))
 
 - thus ensuring that the index of the store is always less than its
   length, wherever it is used, thus making store more secure
  • this requires a change in the signature for all constructors in the following manner
set-ref :: store? (ref? s) storable-value? -> store?
  • eval-ast with store
eval-ast: ast? env? store? -> expressible-value? store?
 (define eval-ast
   (lambda (a e s)
     (cases ast? a
       [num (datum) (values s datum)]
      ;;fill others...
       [id-ref (sym) (values s (lookup-env e sym))]
  • Usage Example
(assume ([x (new-ref 5)])
   (seq
     (setref x 3)
     (deref x 5)))
  • Compare it with C code
{ int x = 5;
  x = 3;
  return x;}

 or

{ int *p;
  p = (malloc(* sizeof(int)));
  *p = 5;
  *p = 3;
  return *p;}
  • what we have implemented using ADT was the second method of implementation of the program in C
  • though the first C code looks more natural, it is not in keeping with the internal implementation ie machine implementation
  • second C code is more like the actual implementation but is difficult to follow with all the pointer notation
  • the ADT implementation gives us a natural approach to the store concept without being too difficult to follow
  • this representation is explicit representation while the former C code is implicit representation

1.4. interpreter for explicit language

  • language supports
    • newref - allocate memory
    • deref - read from memory
    • setref - modify memory
  • ast to include store
ast ::= (newref <ast>) |
        (deref <ast>) |
        (setref <ast1> <ast2>   ; ast1 evaluates to ref & ast2 to
        value
  • instead of being ast try implementing new, deref and setref as primitives of the language……
  • reification cannot be applied to store or environment
  • semantic domain
expressible-value ::= number | boolean | procedure |
                     ref (expressible-value)
denotable-value ::= expressible-value
  • interpreter
(require (prefix store: "store.ss")) 
;every function from store.ss will be prefixed with store:

(define eval-ast
  (lambda (ast e s)
    (cases a ast
      [ num (datum) (values s datum)]
     ...
      
      [newref (a)                                           ;a1
        (let-values ([(s v) (eval-ast a e s)])              ;s2 a1 s
          (match-let ([(list s r) (store:new-ref s v)])     ;s3 s2
            (values s r)))]                                 ;s3

      [deref (a)
        (let-values ([(s r) (eval-ast a e s)])
          (if ((store:ref? s) r)
            (let ([v (store:deref s r)])
              (values s v))
            (error 'eval-ast "not a reference ~a" r)))]

      [setref (a v)
        (let*-values ([(s r) (eval-ast a e s)]
                      [(s v) (eval-ast v env s)])
          (if ((store:ref? s) r)
            (values (store:setref s r v) (void))
            (error 'eval-ast "not a reference ~a" r)))])))
