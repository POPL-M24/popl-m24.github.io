1. Tree data type

(define-datatype tree tree?
  [leaf (val any/c)]
  [int  (val any/c) (left tree?) (right tree?)])

(define leaf.val
  (lambda (t)
    (cases tree t
      [leaf (val) val]
      [else (error 'leaf.val "contract violation:  expected leaf, got ~a" t)])))

(define int.val
  (lambda (t)
    (cases tree t
      [int (val left right) val]
      [else (error 'int.left "contract violation:  expected int, got ~a" t)])))

(define int.left
  (lambda (t)
    (cases tree t
      [int (val left right) left]
      [else (error 'int.left "contract violation:  expected int, got ~a" t)])))

(define int.right
  (lambda (t)
    (cases tree t
      [int (val left right) right]
      [else (error 'int.right "contract violation:  expected int, got ~a" t)])))

2. Recursive functions on trees

2.1. Tree size

(define tree-size
  (lambda (t)
    (cases tree t
      [leaf (v) 1]
      [int (v l r) (+ 1 (tree-size l) (tree-size r))])))

(define tree-height
  (lambda (t)
    (cases tree t
      [leaf (v) 1]
      [int (v l r) (+ 1 (max (tree-height l) (tree-height r)))])))


(define tree-map
  (lambda (t f)
    (letrec ([g (lambda (t)
                  [leaf (v) (f t)]
                  [int (v l r)
                       (int (f v) (g l) (g r))])])
      (g t))))

3. Paths

A direction is either left or right. A path in a tree is a list of direction symbols. Each path identifies a unique position within the tree. Note that a tree defines the set of all legitimate paths.

For example, consider the tree 

(define t (int 1 (leaf 2) (int 3 (leaf 4) (leaf 5))))

The set of paths in t and the (sub)trees identified by each of them is given below: 

() : t
(left) : (leaf 2)
(right) : (int 3 (leaf 4) (leaf 5))
(right left) : (leaf 4)
(right right) : (leaf 5)

;;; paths: tree? -> list-of
(define paths
  (lambda (t)
    (cases tree t
      [leaf (v)
        (list '())]
      [int (v tl tr)
           (append (map (lambda (p) (cons 'left p)) (paths tl))
                   (map (lambda (p) (cons 'right p)) (paths tr)))])))

4. Tree contexts

A tree context for a tree t and a path p is a structure that represents everything surrounding the subtree at position p in t. Another way of thinking of a tree context is that it is the same as tree t except that at position p is a 'hole'. For example, the context (t,p).

5. Contexts as functions

A tree t and a path p in t define a context. The result of filling this context with a tree s is a tree that is identical to t except that it has the tree s at path p. 

  ;;; fill : [t:tree? (path? t) tree?] -> tree?
    (define fill
      (lambda (t p s)  ; t is a tree.  Hole is at position p
                       ; s is a tree used to fill the hole at p.
        (letrec ([fill (lambda (p t)
                         (cond
                           [(null? p) s]
                           [(eq? 'left (first p))
                            (int (int.val t) 
                                 (fill (rest p) (int.left t))
                                 (int.right t))]
                           [else (int
                                   (int.val t)
                                   (int.left t)
                                   (fill (rest p) (int.right t)))]))])
          (fill p t))))

(check-equal? (fill t '() '(leaf 'a)) (leaf 'a) "fill 01")
(check-equal? (fill t '(left) (leaf 'a)) (int 1 (leaf 'a) (int 3 (leaf 4) (leaf 5))) "fill 02")
